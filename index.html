Guía Académica de Laboratorio

Colegio Técnico Profesional de Flores

🎓 Autenticación y Perfiles en la Nube

Esta guía didáctica formal detalla la creación e integración de un sistema unificado de acceso y base de datos utilizando Firebase Authentication y Cloud Firestore en una aplicación híbrida de Ionic y Angular (Standalone Components).

Descargar Código de Referencia (.ZIP) Ver Repositorio en GitHub

Objetivo del Laboratorio

Los estudiantes aprenderán a estructurar pantallas con diseño inmersivo y glassmorphism en SCSS, aprovisionar bases de datos NoSQL seguras y programar lógica asíncrona mediante observables reactivos.

Requisitos del Entorno de Desarrollo

Para garantizar la compatibilidad absoluta con el modelo de Standalone Components y la estructura de archivos provista en esta guía, es fundamental que los estudiantes utilicen versiones modernas y compatibles de Node.js, Angular y el CLI de Ionic.

💡 ¿Por qué es importante verificar la versión de Angular?

A partir de la versión 17, Angular adoptó por defecto la arquitectura Standalone (sin NgModule). Si un estudiante utiliza una versión anterior, los comandos de generación del CLI crearán módulos clásicos en lugar de componentes independientes, provocando incompatibilidades con los códigos de esta guía y diferencias en la nomenclatura de archivos (como auth-guard.ts en lugar de auth.guard.ts).

1. Verificación de Versiones locales

Ejecute los siguientes comandos para verificar las versiones instaladas en la máquina local:

Terminal de comandos 
# Verificar versión de Angular CLI, Node y Sistema Operativo
npx ng version

# Verificar versión de Ionic CLI y entorno híbrido
ionic info

2. Solución: Actualizar Angular y el CLI

Si las herramientas están desactualizadas (versión de Angular CLI menor a 17), la solución para evitar discrepancias es actualizarlas globalmente ejecutando:

Terminal de comandos 
# Actualizar Angular CLI e Ionic CLI a sus últimas versiones globales
npm install -g @angular/cli@latest @ionic/cli@latest

Una vez finalizada la maquetación, los estudiantes aprovisionarán los servicios en la Consola de Google Firebase.

1. Crear el Proyecto

  • Ingrese a la Consola de Firebase.
  • Presione Agregar proyecto y asígnele un nombre identificable (ej. bd-ionic-estudiantes).
  • Desactive la casilla de Google Analytics para acelerar el aprovisionamiento.

2. Configurar Firebase Authentication

  • Vaya a Build > Authentication en el menú lateral y pulse Comenzar.
  • En la pestaña Sign-in method, seleccione el proveedor Correo electrónico/Contraseña.
  • Active la casilla principal del proveedor de identidad y presione Guardar.

3. Habilitar y Configurar Cloud Firestore NoSQL

  • Diríjase a Firestore Database en el menú y haga clic en Crear base de datos.
  • Establezca la ubicación regional física más idónea para su laboratorio.
  • En la pestaña de Reglas (Rules), reemplace las políticas por defecto para autorizar la lectura y escritura únicamente al usuario dueño del perfil:
firestore.rules 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /usuarios/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
  }
}

4. Servicio de Autenticación (`AuthService`)

Genere el servicio inyectable con el CLI:

Terminal de comandos 
npx ionic g service services/auth

Implemente la lógica asíncrona de autenticación y transacciones con la base de datos en Firestore NoSQL:

src/app/services/auth.service.ts 
import { Injectable, inject } from '@angular/core';
import {
  Auth,
  createUserWithEmailAndPassword,
  signInWithEmailAndPassword,
  signOut,
  user,
  User
} from '@angular/fire/auth';
import {
  Firestore,
  doc,
  setDoc,
  getDoc
} from '@angular/fire/firestore';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class AuthService {
  private auth = inject(Auth);
  private firestore = inject(Firestore);

  // Observable reactivo de monitoreo de estado en vivo de la sesión
  user$: Observable = user(this.auth);

  async registrar(email: string, contrasena: string, nombreUsuario: string) {
    try {
      // 1. Registro de credenciales en Auth
      const credenciales = await createUserWithEmailAndPassword(this.auth, email, contrasena);
      const uid = credenciales.user.uid;

      // 2. Expediente NoSQL en Firestore mapeado con su UID
      const docRef = doc(this.firestore, `usuarios/${uid}`);
      await setDoc(docRef, {
        correo: email,
        nombreUsuario: nombreUsuario,
        rol: 'estudiante',
        fechaCreacion: new Date().toISOString()
      });

      return credenciales.user;
    } catch (error) {
      throw error;
    }
  }

  async login(email: string, contrasena: string) {
    try {
      const credenciales = await signInWithEmailAndPassword(this.auth, email, contrasena);
      return credenciales.user;
    } catch (error) {
      throw error;
    }
  }

  async obtenerPerfilUsuario(uid: string) {
    const docRef = doc(this.firestore, `usuarios/${uid}`);
    const snap = await getDoc(docRef);
    return snap.exists() ? snap.data() : null;
  }

  logout() {
    return signOut(this.auth);
  }
}

5. Guardián de Seguridad de Rutas (`AuthGuard`)

Genere el guardián de rutas asíncrono (cuando la terminal te pregunte el tipo de interfaz a implementar, selecciona CanActivate con la barra espaciadora y pulsa Enter):

Terminal de comandos 
npx ionic g guard guards/auth

Configure el interceptor para desviar usuarios no autenticados de regreso a la pantalla de acceso:

src/app/guards/auth.guard.ts 
import { inject } from '@angular/core';
import { CanActivateFn, Router } from '@angular/router';
import { AuthService } from '../services/auth.service';
import { map, take } from 'rxjs/operators';

export const authGuard: CanActivateFn = (route, state) => {
  const authService = inject(AuthService);
  const router = inject(Router);

  return authService.user$.pipe(
    take(1),
    map(user => {
      if (user) {
        return true;
      } else {
        router.navigate(['/login']);
        return false;
      }
    })
  );
};

  formulario!: FormGroup;
  esRegistro: boolean = false;
  verContrasena: boolean = false;

  constructor() {
    addIcons({ mailOutline, lockClosedOutline, personOutline, eyeOutline, eyeOffOutline });
  }

  ngOnInit() {
    this.inicializarFormulario();
  }

  inicializarFormulario() {
    this.formulario = this.fb.group({
      email: ['', [Validators.required, Validators.email]],
      password: ['', [Validators.required, Validators.minLength(6)]],
      nombreUsuario: ['']
    });
  }

  alternarModo() {
    this.esRegistro = !this.esRegistro;
    const controlNombre = this.formulario.get('nombreUsuario');

    if (this.esRegistro) {
      controlNombre?.setValidators([Validators.required, Validators.minLength(3)]);
    } else {
      controlNombre?.clearValidators();
      controlNombre?.setValue('');
    }
    controlNombre?.updateValueAndValidity();
  }

  async enviarFormulario() {
    if (this.formulario.invalid) return;

    const { email, password, nombreUsuario } = this.formulario.value;
    const loading = await this.loadingCtrl.create({
      message: this.esRegistro ? 'Procesando registro...' : 'Validando acceso...',
      spinner: 'crescent'
    });
    await loading.present();

    try {
      if (this.esRegistro) {
        await this.authService.registrar(email, password, nombreUsuario);
        this.mostrarAlerta('Registro Exitoso', 'La cuenta académica ha sido creada con éxito.');
      } else {
        await this.authService.login(email, password);
      }
      this.router.navigate(['/home']);
    } catch (error: any) {
      this.mostrarAlerta('Autenticación Fallida', this.traducirError(error.code));
    } finally {
      loading.dismiss();
    }
  }

  private async mostrarAlerta(titulo: string, mensaje: string) {
    const alert = await this.alertCtrl.create({
      header: titulo,
      message: mensaje,
      buttons: ['Aceptar']
    });
    await alert.present();
  }

  private traducirError(codigo: string): string {
    switch (codigo) {
      case 'auth/invalid-credential':
        return 'Las credenciales ingresadas son incorrectas o no están registradas.';
      case 'auth/email-already-in-use':
        return 'El correo electrónico ya se encuentra registrado.';
      case 'auth/weak-password':
        return 'La seguridad de la contraseña es deficiente (ingrese mínimo 6 caracteres).';
      case 'auth/invalid-email':
        return 'El formato de correo ingresado no cuenta con un formato válido.';
      case 'permission-denied':
      case 'firestore/permission-denied':
        return 'Error de privilegios. Asegúrese de haber configurado las Reglas de Seguridad en Cloud Firestore.';
      case 'unavailable':
        return 'No es posible conectar con Firestore. Habilite el servicio de base de datos.';
      default:
        return `Error del sistema: ${codigo}. Intente de nuevo.`;
    }
  }
}

Fase IV: Pruebas de Integración y Resolución de Errores

Instruya a los estudiantes en el laboratorio escolar para llevar a cabo el protocolo de verificación.

1. Protocolo de Pruebas de Software

  1. Arranque el servidor local en la consola: npx ionic serve.
  2. Navegue directamente a la url: http://localhost:8100/home. Valide que el navegador bloquee el acceso y lo redirija de inmediato a la ruta /login.
  3. Pulse la alternancia a modo Registrarse, rellene los campos e introduzca un correo de pruebas. Presione enviar.
  4. Acepte la confirmación emergente de la alerta y asegúrese de que la aplicación lo encamine automáticamente hacia el Dashboard con sus datos de perfil debidamente desplegados en pantalla.

2. Diagnóstico Común: Reglas NoSQL

🛑 El usuario se crea en Authentication pero muestra "Error de permisos..."

Explicación: El motor de Firebase Auth completó satisfactoriamente el primer paso del registro escolar, pero al intentar transaccionar el Nombre de Usuario con Cloud Firestore Database en el backend, la conexión es declinada debido a la falta de políticas de base de datos o por no haberse inicializado el recurso.

Solución: Ingrese a la consola del proyecto en Firebase, diríjase a Firestore Database, valide que la base de datos se haya creado y verifique en la pestaña de Rules (Reglas) que haya publicado la regla de escritura para usuarios autenticados provista en la Fase II.
